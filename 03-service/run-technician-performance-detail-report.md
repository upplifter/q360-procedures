# Run the Technician Performance Detail Report

## What This Procedure Does

The Technician Performance Detail Report shows individual service calls for each technician with dispatch information, problem/solution descriptions, and performance metrics. Use this report for quality assurance, training evaluation, and individual performance feedback.

## Before You Begin

- Have permission to access Service Reports
- Determine the date range for evaluation
- Identify which technician(s) to analyze
- Prepare performance standards for comparison

## Steps

### Step 1: Open Technician Performance Detail Report

Go to Service > Reports > Technician Performance Detail. The Print Report dialog opens with filter options.

### Step 2: Set Filter Criteria

Configure filters to define the analysis:

| Field | What to select |
|-------|----------------|
| Company | Select the company |
| Branch | Select the branch (optional) |
| Department | Select the department (optional) |
| Date Range | Enter start and end dates |
| Technician | Select specific technician (optional) |

Leave Technician blank to include all technicians, or select one for focused evaluation.

### Step 3: Run the Report

Click Run Report. The report generates and displays in Print Report dialog or Live Data grid.

### Step 4: Review Technician Call Details

The report displays line items showing:

- Technician Name
- Call No
- Customer
- Site
- Product
- Dispatch Date/Time
- Arrival Date/Time
- Call Completion Time
- Problem Description
- Solution Description
- Response Time
- Call Duration
- Call Outcome (First Call Fix, On-site, Escalated, etc.)
- Customer Satisfaction Rating (if available)

### Step 5: Evaluate Call Quality and Outcomes

Review problem and solution descriptions for clarity and thoroughness. Assess first-call fix rate. Check response times against commitments. Review customer satisfaction ratings. Identify patterns in call types handled and efficiency metrics.

## What Happens Next

Results support individual performance feedback and coaching. Use call details for quality assurance, training planning, and recognition. Share results with technicians for development and improvement discussions. Track progress by comparing multiple periods.

## Common Issues

**No Data Displayed**
Verify the date range includes calls with technician assignments. Ensure the technician filter is not too restrictive. Check that all calls have proper technician assignment.

**Problem or Solution Description Missing**
Verify technicians are completing all required fields on call records. Review data entry training to improve completeness. Investigate specific calls with missing information.

**Cannot Verify First Call Fix Rate**
Ensure call records include outcome codes. Verify follow-up calls are properly linked to original calls. Check if your system tracks first-contact resolution metrics.
