# Run the Technicians with Work Orders Report

## What This Procedure Does

The Technicians with Work Orders Report lists work orders with associated service call, dispatch, customer, and technician information. Use this report for workload planning, technician allocation analysis, and field team coordination.

## Before You Begin

- Have permission to access Service Reports
- Determine the date range for analysis
- Identify the company and branch scope
- Understand your work order types and categories

## Steps

### Step 1: Open Technicians with Work Orders Report

Go to Service > Reports > Technicians with Work Orders. The Print Report dialog opens with filter options.

### Step 2: Set Filter Criteria

Configure filters to define the analysis:

| Field | What to select |
|-------|----------------|
| Company | Select the company |
| Branch | Select the branch (optional) |
| Department | Select the department (optional) |
| Date Range | Enter start and end dates |

### Step 3: Run the Report

Click Run Report. The report generates and displays in Print Report dialog or Live Data grid.

### Step 4: Review Work Order Assignments

The report displays work order records showing:

- Work Order No
- Service Call No (linked call)
- Technician Name
- Customer Name
- Site Location
- Work Order Description
- Work Order Status
- Dispatch Date
- Promised Completion Date
- Actual Completion Date
- Hours Allocated
- Hours Actual

### Step 5: Analyze Workload Distribution

Review total work orders and hours by technician. Identify technicians with over-capacity or under-capacity. Check if promised completion dates are being met. Evaluate actual hours versus estimated hours.

## What Happens Next

Results guide technician scheduling and workload balancing. Identify if workload is fairly distributed. Plan training or hiring if capacity issues exist. Use the report to support resource planning and field team optimization.

## Common Issues

**No Work Orders Displayed**
Verify work orders are created and linked to service calls. Check that the date range includes active work orders. Ensure the company and branch filters are correct.

**Technician Not Listed**
Confirm the technician is assigned to work orders in the system. Check if technician records are active. Verify the date range includes work orders for the technician.

**Hours Mismatch**
Compare estimated hours on work order to actual time logged. Investigate specific work orders with large variances. Review if technicians are completing time entries accurately.
