# Run the Project Bookings / Revenue / Backlog Report

## What This Procedure Does

This procedure shows you how to run the Project Bookings / Revenue / Backlog Report. The report displays project bookings, revenue, and backlog information in one view, helping you track financial activity across projects.

## Before You Begin

Ensure you have access to the Projects module and the Reports section. You should understand your organization's project structure and which projects you need to review.

## Steps

### Step 1: Navigate to the Report

From the Main Menu, click **Projects**. In the Projects menu, click **Reports**. Select **Project Bookings/Revenue/Backlog** from the report list.

### Step 2: Set Filter Criteria

The report filter panel opens with fields for narrowing down your data. Set the filters using the table below:

| Field | What to enter |
|-------|---------------|
| Project | Select specific projects or leave blank to include all |
| Date Range | Enter the start and end dates for the period you want to review |
| Status | Choose Active, Closed, or All projects |
| Division | Select a division or leave blank for all divisions |

Click **Apply Filters** to update the report with your selections.

### Step 3: Run the Report

Click the **Run** or **Refresh** button to execute the report with your chosen filters.

### Step 4: Review the Report Output

The report displays bookings, revenue, and backlog data in columns. Review the totals at the bottom to verify the data matches your expectations. Examine individual project rows to identify any anomalies or trends.

### Step 5: Export or Print (Optional)

To save the report, click **Export** and choose your format (Excel, PDF, or CSV). To print, click **Print** and follow your printer settings.

## What Happens Next

The report data refreshes based on your filters and displays current project financial information. You can use this data for financial analysis, forecasting, and project performance reviews. Save or print the report for your records or to share with stakeholders.

## Common Issues

**Report shows no data**
If the report displays empty results, verify your filter criteria. Check that the date range includes active projects and that you have selected appropriate project statuses. Adjust the filters and run again.

**Unexpected revenue or backlog amounts**
Review the date range selected in your filters. Revenue and backlog are period-specific, so selecting a different date range may explain differences. Confirm that the correct projects are included in your filter.

**Export button is not working**
Ensure you have export permissions in your user role. If the export button remains inactive, try refreshing the report or closing and reopening the report screen. Contact your system administrator if the issue persists.

**Report takes a long time to load**
Large date ranges or multiple project selections can slow report performance. Narrow your filter criteria by selecting fewer projects or a shorter time period. Run the report again after adjusting your filters.

**Column headers are truncated or unclear**
Widen the report columns by clicking and dragging the column borders. This allows you to see full header names and data values more clearly.
