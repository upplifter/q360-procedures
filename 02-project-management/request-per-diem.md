# Request Per Diem

## What This Procedure Does

This procedure submits a per diem expense request for a project team member who is traveling on project-related work. Per diem requests cover daily allowances for meals and incidental expenses during travel. Once submitted, the per diem request goes through an approval workflow.

## Before You Begin

- Verify you have EXPENSE (Create) permission
- Identify the project or task associated with the travel
- Gather traveler information including travel dates and per diem amounts

## Steps

### Step 1: Look Up the Project or Task

Navigate to the project or task for which the per diem is being requested. You can access projects through Main Menu > Projects > Projects. If requesting per diem for a specific task, you can also access it from My Tasks.

### Step 2: Open the Per Diem Request Form

Click the Extended Menu (3 vertical dots) and select Request Per Diem from the available options.

### Step 3: Fill in the Per Diem Request Form

Complete the following fields in the per diem request form:

| Field | What to enter |
|-------|---------------|
| Traveler Name | Name of the employee requesting per diem |
| Travel Dates | Start and end dates of the travel period |
| Amount | Per diem amount claimed (daily or total, as per company policy) |
| Description | Purpose of travel and relevant details about the project work |
| Destination | City or location of travel |

### Step 4: Submit the Request

Click the Submit button. Q360 creates an expense item for the selected traveler and submits it into the approval workflow. The traveler and project manager receive notification of the submitted request.

## What Happens Next

The expense item enters the approval workflow and is routed to the appropriate approver based on company policy. Once approved, the per diem can be posted and reimbursed. The expense appears in project expense reports and contributes to project costs. If rejected, the traveler is notified and can revise and resubmit the request.

## Common Issues

**Extended Menu doesn't show the Request Per Diem option**
Verify you have EXPENSE (Create) permission. Confirm you are on a valid project or task record, not just viewing a list. Some users may have limited permissions that restrict per diem requests; contact your system administrator if the option remains unavailable.

**The traveler name field doesn't populate with the correct employee**
Ensure the traveler exists in the system as an employee or resource. If the traveler is new to the system, they may need to be added to the resource list before per diem requests can be created. Type the full name as it appears in the employee directory.

**Cannot submit the form due to validation errors**
Verify all required fields are completed. Check that travel dates are in the correct format (typically MM/DD/YYYY). Confirm the per diem amount is a valid number. Some fields may have specific formatting requirements; review any error messages for guidance.

**Submitted per diem doesn't appear in the approval queue**
Refresh the form or check the expense module directly to confirm submission. Submitted expenses may take a few moments to appear in the workflow. Verify the approver has access to the Projects expense review area.

**Need to modify a submitted per diem request**
If the request has not yet been approved, you may be able to recall and edit it depending on system configuration. Contact your approver or system administrator for assistance in modifying submitted requests. Once approved, you typically cannot edit the expense directly.
