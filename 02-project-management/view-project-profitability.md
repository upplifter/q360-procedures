# View Project Profitability

## What This Procedure Does

This procedure enables you to access and review the financial performance of a project through the Profit tab. You can view comprehensive financial figures including projected revenue, projected cost, actual revenue, actual cost, earned values, and profit margins.

## Before You Begin

- Verify you have PROJECT (View) permission assigned to your role
- Have the project number or name available for lookup
- Be prepared to review financial metrics and performance indicators

## Steps

### Step 1: Look Up the Project
Navigate to the Projects module and locate the project you wish to review using the search or filter options.

### Step 2: Access the Profit Tab
Click on the Profit tab to display the project's financial performance information.

### Step 3: Review Available Financial Figures
The Profit tab displays comprehensive financial data for the project, including:
- Projected revenue
- Projected cost
- Actual revenue
- Actual cost
- Earned values
- Profit margins

Review these figures to assess the project's financial performance against plan and to monitor profitability trends.

### Step 4: Reference Detailed Profit Information
For detailed explanations of the various numbers and calculations displayed in the Profit tab, search for "Q360 Demonstration Project Profit" in Q360 > Help > Contents.

## What Happens Next

You now have visibility into the project's financial performance. Use this information to monitor profitability, identify variances from plan, and inform project management decisions. Financial data in the Profit tab updates as time entries, material receipts, and SOV progress are recorded.

## Common Issues

**The Profit tab does not appear on the project form.**
Verify that you have PROJECT (View) permission assigned to your user role. If you have the required permission but the tab is not visible, refresh your browser to reload the project form. Contact your system administrator if the tab remains unavailable.

**Financial figures appear to be zero or blank.**
Project profitability data populates as project activity is recorded. If the project has no time entries, material receipts, or SOV progress recorded, financial figures may be zero or blank. Verify that the project has active work recorded before reviewing profitability metrics.

**Actual cost figures do not match my expectations.**
Verify that all relevant time entries, material receipts, and purchase orders have been posted to the project. Unposted transactions are not included in actual cost calculations. Review the Profit tab after posting all pending project transactions.

**Projected revenue or cost figures appear incorrect.**
Verify that the SOV or project estimates are accurately entered with correct quantities and amounts. Projected figures are based on SOV line items or project estimates. Review the SOV tab (see PROJ-SOV-001) or project estimates to confirm accuracy.

**The Help content for Q360 Demonstration Project Profit is not accessible.**
Verify that you have access to the Q360 Help system. Navigate to Q360 > Help > Contents and search for the topic. If Help content is unavailable, contact your system administrator to verify that Help documentation is installed and accessible in your environment.
